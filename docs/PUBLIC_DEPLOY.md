# Sharing a public demo link (free)

Recommended stack: **Vercel (app) + Neon (Postgres)**. Both free, both no-credit-card to start.

- **Vercel** hosts the Next.js app and gives a public https URL on every push. The same responsive
  app is the mobile-web experience — open the link on a phone.
- **Neon** is free Postgres that does NOT pause after inactivity (Supabase's free tier pauses
  after 7 days, which would break a shared demo link). Neon adds only a ~0.5s cold start.

The orchestrator generates `vercel.json`, the seed loader, and a labeled **demo-logins** file so
anyone opening your link can sign in as a fan, helper, moderator, business, or admin (all use
password `roarpass-demo`, synthetic data only).

It does NOT log in for you. Account creation, `vercel login`, and entering the Neon connection
string are one-time manual steps you perform — full copy-paste walkthrough in the product repo's
`docs/DEPLOY_PUBLIC.md` (generated by `scripts/deploy_vercel.py`).

## Native mobile apps
Can't be "freely deployed to a public link": the App Store / Play Store need paid developer
accounts and review. The mobile-web version on the Vercel link covers phone demos; native is
deferred as scoped.

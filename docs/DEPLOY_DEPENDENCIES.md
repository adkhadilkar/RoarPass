## 1. Runtime Prerequisites

Install these on your **host machine** before running `scripts/deploy_local.sh`.

| Tool | Minimum Version | Why |
|------|-----------------|-----|
| Docker Engine | 24.0 | Container runtime |
| Docker Compose (plugin) | 2.24 | Orchestration |
| Node.js | 20 LTS | Frontend build & dev server |
| pnpm | 8.x | JS package manager (lockfile pinned) |
| Python | 3.11+ | Backend API |
| uv _(or pip + venv)_ | latest | Python package manager |
| Git | 2.40+ | Source control / hooks |
| mkcert | 1.4+ | Local TLS (optional but recommended) |
| jq | 1.6+ | Smoke-check JSON parsing |
| curl | 7.86+ | HTTP probes |
| GNU parallel | 20230522+ | Parallel smoke checks |

### Quick install (macOS — Homebrew)

```bash
brew install docker node pnpm python@3.11 uv mkcert jq curl parallel
```

### Quick install (Ubuntu/Debian)

```bash
# Docker — follow https://docs.docker.com/engine/install/ubuntu/ (manual action, see §7)
sudo apt-get install -y nodejs npm python3.11 python3-pip jq curl parallel
npm install -g pnpm
pip install uv
# mkcert — https://github.com/FiloSottile/mkcert#linux (manual action, see §7)
```

> **⚠ Windows**: Use WSL 2 with the Ubuntu path above. Docker Desktop for Windows is
> supported but requires manual license acknowledgement (see §7).

## 7. Manual Actions Required (Human Attention)

> The setup script (`scripts/deploy_local.sh`) will **pause and prompt** at each of these.
> It will **not** perform them silently or automatically.

### M-1 · Docker Engine Installation

Docker Engine requires accepting the [Docker Subscription Service Agreement](https://www.docker.com/legal/docker-subscription-service-agreement/).
**Docker Desktop** additionally requires a paid licence for organisations with >250 employees
or >$10 M revenue.

**Action**: Install Docker manually following the official docs for your OS, and confirm
you have accepted the relevant licence before proceeding.

### M-2 · mkcert Local CA Installation

Running `mkcert -install` installs a root CA into your system/browser trust stores.
This is a **privileged security-setting change** the script will not do automatically.

**Action**: Run `mkcert -install` yourself after reviewing what it does:
<https://github.com/FiloSottile/mkcert#installation>

### M-3 · Stripe CLI (optional)

The Stripe CLI requires creating a Stripe account and authenticating via `stripe login`
(OAuth browser flow). The script does not automate this.

**Action**: `stripe login` — follow the browser prompt. Required only for testing
payment webhooks locally.

### M-4 · Environment Secrets

Secrets must be populated in `.env.local` (frontend) and `backend/.env` (API) before
first run. The script copies `.env.example` files but **does not fill in real secrets**.

Variables requiring human input:

| Variable | Service | Notes |
|----------|---------|-------|
| `STRIPE_SECRET_KEY` | backend | From Stripe dashboard |
| `STRIPE_WEBHOOK_SECRET` | backend | From `stripe listen` output |
| `STRIPE_PUBLISHABLE_KEY` | frontend | From Stripe dashboard |
| `SENTRY_DSN` | backend + frontend | From Sentry project |
| `MAPLIBRE_STYLE_URL` | frontend | Map tile provider URL |
| `NEXT_PUBLIC_MAPLIBRE_STYLE_URL` | frontend | Same, exposed to browser |
| `SMTP_HOST` / `SMTP_PORT` | backend | Production SMTP (MailHog used locally) |
| `JWT_SECRET_KEY` | backend | Generate: `openssl rand -hex 32` |
| `ENCRYPTION_KEY` | backend | AES key for PII at rest |

### M-5 · GDPR / CCPA Data Processing Agreement

If you load real user data into your local environment, ensure you have a valid
legal basis under GDPR Art. 6 and/or CCPA. The sample seed data (`scripts/seed_db.py`)
uses synthetic data only.

### M-6 · Node.js / Python Version Switching

If you have multiple versions installed, ensure the correct versions are active
(`node --version`, `python3 --version`). The script checks but does **not** switch
versions via `nvm`/`pyenv` automatically (that would silently change your shell state).

## 9. Environment Variables Reference

Full reference lives in `backend/.env.example` and `frontend/.env.example`.
Key groupings:

```
# Database
DATABASE_URL=postgresql+asyncpg://roarpass:roarpass@localhost:5432/roarpass

# Redis
REDIS_URL=redis://localhost:6379/0

# Search
MEILISEARCH_URL=http://localhost:7700
MEILISEARCH_MASTER_KEY=<generate with: openssl rand -hex 16>

# Storage
S3_ENDPOINT_URL=http://localhost:9000
S3_ACCESS_KEY_ID=roarpass-local
S3_SECRET_ACCESS_KEY=roarpass-local-secret
S3_BUCKET_NAME=roarpass-media

# Auth
JWT_SECRET_KEY=<generate with: openssl rand -hex 32>
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=30

# Stripe (local — use test keys)
STRIPE_SECRET_KEY=sk_test_...
STRIPE_PUBLISHABLE_KEY=pk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Email (local MailHog)
SMTP_HOST=localhost
SMTP_PORT=1025
SMTP_TLS=false
EMAIL_FROM=noreply@roarpass.local

# App
APP_ENV=local
LOG_LEVEL=debug
ALLOWED_ORIGINS=http://localhost:3000,https://localhost:3000
```
